# 実装計画

本ディレクトリでは、Horus バックエンドの機能拡張における実装計画を Markdown 形式で管理します。

## ディレクトリ構成

```
implementation-plans/
├── README.md           # 本ファイル（概要と運用方針）
└── {feature-name}.md   # 各機能の実装計画
```

## 設計ドキュメントとの関係

- **設計ドキュメント** (`docs/design/`): 機能の仕様・アーキテクチャ・技術方針を定義
- **実装計画** (`docs/implementation-plans/`): 設計に基づく具体的な実装ステップ・タスク・進捗を管理

実装計画は設計ドキュメントを参照し、Phase 別のタスク分解と進捗追跡を行います。

## 実装計画ドキュメントのフォーマット

各実装計画（`{feature-name}.md`）には以下を含めることを推奨します：

1. **概要**: 機能の要約と設計ドキュメントへのリンク
2. **前提条件**: 依存する機能・ブランチ・作業完了項目
3. **Phase 一覧**: 実装フェーズとタスクの分解
4. **進捗チェックリスト**: 各タスクの完了状況
5. **変更履歴**: 計画の更新記録

## 運用方針

- 新機能の実装を開始する前に、本ディレクトリに実装計画を作成する
- 設計ドキュメントが `docs/design/` に存在する場合は、実装計画から参照する
- 実装着手時は、計画に対応するブランチを切ることを推奨（例: `feature/flight-plan`）
